# ComfyUI 中文版快速开始指南

## 一、前言

ComfyUI 是一款基于节点式工作流的 AI 图像生成工具，以灵活、高效、可定制的特点深受用户喜爱，尤其适合需要精细控制生成流程的创作者。本指南为中文版适配版本，将帮助你快速完成 ComfyUI 的环境搭建、首次启动，并掌握核心基础操作，轻松开启 AI 绘图之旅。

## 二、前期准备

### 2.1 硬件要求

- 显卡：推荐 NVIDIA 显卡（显存 ≥6GB，8GB 及以上更佳），支持 CUDA 加速；AMD 显卡需额外配置 ROCm 环境，兼容性稍差。

- 内存：≥16GB（生成高清图像或复杂工作流时，32GB 内存体验更好）。

- 存储：预留至少 20GB 空闲空间，用于安装软件、存放模型文件（模型文件通常较大，单个基础模型约 2-8GB）。

### 2.2 软件与环境依赖

- 操作系统：Windows 10/11（推荐）、Linux（Ubuntu 20.04 及以上）。

- 必备工具：

  - Python 3.10.x（推荐 3.10.6，避免使用 3.11 及以上版本，可能存在兼容性问题）。

  - Git（用于克隆 ComfyUI 仓库，可选，也可直接下载压缩包）。

  - NVIDIA 显卡驱动：确保驱动版本 ≥528.02（支持 CUDA 12.0 及以上）。

## 三、ComfyUI 安装与启动

### 3.1 下载 ComfyUI

便携版（Windows 用户推荐，快速上手）
访问[ComfyUI 官方网站](https://www.comfy.org/zh-cn/download)，找到“便携版”下载入口，下载后直接解压到本地目录（如 D:\Apps\ComfyUI），无需额外安装依赖，解压即可使用。

### 3.2 安装依赖（源码版/压缩包版）

1. 进入 ComfyUI 安装目录（例如 Windows：D:\ComfyUI；Linux：~/ComfyUI）。

2. Windows 用户：双击运行“requirements.txt”同级目录下的“install_windows.bat”批处理文件，程序将自动安装所需依赖（包括 PyTorch 等核心库）。

3. Linux 用户：在终端中进入 ComfyUI 目录，输入命令：pip install -r requirements.txt，手动安装依赖。

4. 中文用户优化：若安装依赖时网络较慢，可使用国内镜像源加速，命令示例：

```bash
$ pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 3.3 下载和放置模型文件

ComfyUI 默认不包含 AI 模型，需手动下载并放置到对应目录：

1. 下载基础模型（如 Stable Diffusion 1.5、SDXL 1.0、通义万相等），推荐从[Civitai](https://civitai.com/models)、[Hugging Face](https://huggingface.co/models)、[魔搭社区](https://www.modelscope.cn/models)等平台下载。

2. 将下载的模型文件（.safetensors 或.ckpt 格式）复制到 ComfyUI 目录下的“models/checkpoints”文件夹中。

3. （可选）如需使用 LoRA、ControlNet 等功能，分别将对应模型放置到“models/lora”“models/controlnet”目录；若使用通义万相相关模型，需按模型说明放置到对应子目录。

### 3.4 启动 ComfyUI

1. Windows 用户：

   - 源码版/压缩包版：双击运行“run_nvidia_gpu.bat”（NVIDIA 显卡，推荐）、“run_nvidia_gpu_fast_fp16_accumulation.bat”（NVIDIA 显卡，显存较小用户）或“run_cpu.bat”（无显卡，速度极慢）。

   - 便携版：直接进入解压目录，双击运行对应显卡的启动脚本即可。

# 四、界面基础认知（中文版）

ComfyUI 中文版界面核心分为 4 个区域，新手需重点熟悉：

### 4.1 节点面板（左侧）

包含所有可用功能节点，按类别划分（如加载器、采样器、条件控制、工具等）：

- 加载器（原 Loaders）：用于加载模型、VAE、CLIP 等核心组件。

- 采样器（原 Sampling）：核心生成模块，控制采样器类型、步数、分辨率等参数。

- 条件控制（原 Conditioning）：用于输入正向提示词、反向提示词。

- 工具（原 Utility）：辅助功能，如图像保存、种子设置、参数调整等。

### 4.2 工作流画布（中间）

核心操作区域，用于拖拽节点、连接节点形成生成流程。空白处右键可快速搜索并添加节点，选中节点后按 Delete 键可删除。

### 4.3 参数面板（右侧）

选中某个节点后，此处会显示该节点的可调节参数（如模型选择、提示词输入、采样步数、图像尺寸等），可直接在此修改参数。

### 4.4 结果面板（底部）

显示图像生成进度和最终生成结果，可点击结果图像查看大图、下载图像（右键保存），也可查看生成日志（用于排查错误）。

## 五、基础工作流实操（生成第一张图）

下面通过“提示词生成图像”的基础工作流，带你快速体验生成过程（所有节点均使用中文版名称）：

1. 添加“模型加载器”节点（原 Checkpoint Loader）：

   - 左侧节点面板找到“加载器 → 模型加载器”，拖拽到画布中。

   - 选中该节点，右侧参数面板点击“模型名称（ckpt_name）”下拉框，选择已放置在“models/checkpoints”目录下的模型（如“v1-5-pruned-emaonly.safetensors”）。

2. 添加“正向提示词编码”和“反向提示词编码”节点（原 CLIP Text Encode）：

   - 左侧节点面板找到“条件控制 → CLIP 文本编码（正向）”和“CLIP 文本编码（反向）”，拖拽到画布。

   - 分别选中两个节点，在右侧参数面板的“文本（text）”框中输入提示词：
     正向提示词：一位美丽的女孩，日落，黄金时刻，精致的脸庞，柔和光线（示例，可自定义）。反向提示词：丑陋，模糊，低质量，手部畸形（用于排除不好的效果）。

   - 将“模型加载器”节点的“CLIP”输出端，分别连接到两个“CLIP 文本编码”节点的“clip”输入端（点击输出端拖动到输入端即可完成连接）。

3. 添加“自定义采样器”节点（原 SamplerCustom，核心生成节点）：

   - 左侧节点面板找到“采样器 → 自定义采样器”，拖拽到画布。

   - 连接节点： - 将“模型加载器”的“模型（model）”输出端连接到“自定义采样器”的“模型（model）”输入端。- 将“正向提示词编码”的“条件（conditioning）”输出端连接到“自定义采样器”的“正向（positive）”输入端。- 将“反向提示词编码”的“条件（conditioning）”输出端连接到“自定义采样器”的“反向（negative）”输入端。

   - 设置采样参数（右侧面板）： - 采样器名称（sampler_name）：选择采样器（新手推荐“euler_a”，速度快；追求质量可选“DPM++ 2M Karras”）。- 步数（steps）：采样步数（推荐 20-30 步，步数越多质量越高但速度越慢）。- 宽度/高度（width/height）：图像分辨率（新手推荐 512×768 或 768×512，避免过大导致显存不足）。- 种子（seed）：随机种子（-1 表示随机，固定种子可生成相同风格的图像）。

4. 添加“保存图像”节点（原 Save Image）：

   - 左侧节点面板找到“工具 → 保存图像”，拖拽到画布。

   - 将“自定义采样器”的“图像（image）”输出端连接到“保存图像”的“图像（image）”输入端。

   - （可选）在右侧参数面板设置“文件名前缀（filename_prefix）”和“保存路径（folder_path）”。

5. 执行生成：

   - 点击“自定义采样器”节点右上角的“队列（Queue）”按钮（或画布空白处右键选择“队列提示词（Queue Prompt）”）。

   - 底部结果面板显示生成进度，进度条完成后，即可在结果面板看到生成的图像，右键可保存到本地。

## 六、常见问题与解决办法

### 6.1 启动失败，提示“缺少 xxx 模块”

原因：依赖未安装完整。解决办法：重新运行安装脚本（Windows）或执行“pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple”（国内镜像源加速），确保网络稳定，若某模块安装失败，可手动执行“pip install 模块名 -i 镜像源地址”单独安装。

### 6.2 生成时提示“显存不足（Out of memory）”

解决办法：

- 降低图像分辨率（如从 1024×1024 改为 512×512）。

- 减少采样步数（如从 50 步改为 25 步）。

- 关闭其他占用显存的程序（如游戏、其他 AI 工具）。

- 使用“FP16”精度（部分模型支持，可在模型加载器节点设置）；NVIDIA 显卡用户可尝试运行“run_nvidia_gpu_fast_fp16_accumulation.bat”启动脚本，降低显存占用。

### 6.3 浏览器无法访问 ComfyUI 界面

原因：端口被占用或启动失败。解决办法：

- 检查启动日志，确认是否有“端口 8188 被占用（Port 8188 is in use）”提示，若有，关闭占用端口的程序，或修改 ComfyUI 启动端口（修改 main.py 中的“port=8188”为其他端口，如 8189）。

- 确认启动命令正确，NVIDIA 显卡需运行“run_nvidia_gpu.bat”，避免误运行 CPU 版本脚本。

### 6.4 生成图像模糊、质量差

解决办法：

- 提高采样步数（建议 ≥25 步）。

- 更换更高质量的模型（如 SDXL 模型比 SD 1.5 模型质量更高，通义万相模型支持中英双语渲染，质量优异）。

- 优化提示词，增加细节描述（如“高度细节，8k，超高清”）。

- 调整采样器（推荐“DPM++ 2M Karras”“Euler a”）。

### 6.5 中文提示词生成效果差

原因：部分模型对中文支持不佳。解决办法：更换支持中英双语的模型（如通义万相、Z-Image-Turbo 等）；若使用其他模型，可尝试在中文提示词后补充英文关键词辅助生成。

## 七、进阶学习方向

- 学习使用 ControlNet（姿态控制网络）：实现图像姿态、轮廓、深度等精准控制。

- 掌握 LoRA 加载：通过 LoRA 快速切换风格、添加特定元素，如动漫风格、特定人物特征等。

- 自定义工作流：组合多个节点实现复杂效果（如图生图、超分、去模糊、视频生成等）。

- 模型优化：学习使用模型量化、FP16/FP8 精度等，平衡质量与速度。

- 参考中文社区资源：[魔搭社区](https://www.modelscope.cn/models)（模型下载）、B 站（实操教程）、Civitai 中文区（工作流分享）、[ComfyUI 官方文档中文版](https://docs.comfy.org/zh-CN)。

## 八、总结

ComfyUI 的核心优势在于节点式工作流的灵活性，中文用户入门关键是先掌握“加载模型 → 设置提示词 → 采样生成 → 保存图像”的基础流程，再逐步探索进阶功能。遇到问题时，优先查看启动日志和中文社区资源，多尝试调整参数、更换模型，就能快速提升使用熟练度。祝你在 ComfyUI 的创作之旅中顺利！
